Wireshark and the basic functionalities

Wireshark is a network analysis tool. It captures packets in real time and displays them in a human-readable format. It can also be used to view packet captures that are already saved for analysis. Wireshark uses filters, color-coding, and other features that help with the analysis of network traffic and inspect individual packets.

First, opened a pcap file in Wireshark. In this case, the pcap file is RADIUS.pcap

We can see the network traffic that has been captured before and stored. We can see the time, source and destination IPs, the protocol used, the lengths of the packets along with other information inside the packets.

To capture network traffic in real time:

Here we see the network interface connected to the machine. Double-clicked and opened the network interface Ethernet 3

It shows us all the packets labeling each one with it’s protocol and the hosts it’s travelling between as well. It can be minized such that we don’t see all this traffic but only the filtered needed packets. For this we use display filter to filter the traffic as we want. Traffic can be filtered based on source ip-address, the ports used, the protocol used, or even the MAC address of the host sending this traffic.

To filter for DNS traffic:

Typed dns on the searchbar above and hit enter

A few packets are shown as a filtered result.

To generate more DNS traffic, the ping command can be used on the command prompt.

Now a lot of DNS traffic is shown flowing through the network interface.

The red button on top is clicked to stop capturing any more traffic. From the options menu: file > close to close the file. One can choose to save or discard the filtered results by following the prompt window that pops-up next. Also cleared the display filter applied before.
